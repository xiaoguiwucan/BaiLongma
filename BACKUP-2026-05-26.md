# BaiLongma 当前版本备份与 Mac 自部署说明

备份日期：2026-05-26
当前维护仓库：[xiaoguiwucan/BaiLongma](https://github.com/xiaoguiwucan/BaiLongma)
原始上游仓库：[xiaoyuanda666-ship-it/BaiLongma](https://github.com/xiaoyuanda666-ship-it/BaiLongma)
当前应用版本：`2.1.222`
推荐运行方式：Mac 本地源码运行 Electron 桌面版

这份文档用于记录当前这版白龙马的代码状态、语音能力改动、Git/GitHub 备份方法，以及在另一台 Mac 上从零恢复部署的方法。

## 0. 版本记录要求

从 `v2.1.209` 开始，每次版本备份都必须同步更新：

- `CHANGELOG.md`：写清版本号、日期、更新内容、改变原因、影响范围、部署注意事项。
- `README.md`：更新当前版本号，并在“版本更新记录”里补摘要。
- Brain UI 设置页：在“更新 -> 更新说明”里补最近版本摘要。
- `BACKUP-YYYY-MM-DD.md`：写清当前备份版本、Git tag、哪些文件上传、哪些文件留在本地。
- `package.json` 和 `package-lock.json`：版本号必须一致。
- GitHub Release：每个版本都必须发布 Release，Release 正文必须写清更新内容、改变原因、部署方法、附件说明和已知限制。
- Release 附件：需要上传该版本源码包和 Git bundle；如有安装包，也要一并上传。

备份不是只提交代码，也要让以后的人看得懂这个版本为什么存在、改了哪里、怎么恢复。

## 5. Mac 从零部署

### 5.1 系统准备

建议环境：

- macOS Apple Silicon 或 Intel Mac
- Node.js 20 LTS 或更高
- Python 3.11
- Git
- Xcode Command Line Tools

安装 Xcode Command Line Tools：

```bash
xcode-select --install
```

确认版本：

```bash
node -v
npm -v
python3.11 --version
git --version
```

如果没有 Python 3.11，可用 Homebrew 安装：

```bash
brew install python@3.11
```

### 5.2 拉取项目

```bash
cd ~/开发项目
git clone https://github.com/xiaoguiwucan/BaiLongma.git
cd BaiLongma
```

如果要恢复本次备份点：

```bash
git fetch --tags
git checkout backup-2026-05-26-local-voice
```

如果希望继续在最新主分支开发：

```bash
git checkout main
git pull origin main
```

### 5.3 安装 Node 依赖

```bash
npm install
```

如果 `better-sqlite3` 或 Electron 原生依赖报错，可尝试：

```bash
npm rebuild
npx electron-rebuild -f -w better-sqlite3
```

### 5.4 配置 LLM API Key

创建 `.env`：

```bash
touch .env
```

最小示例：

```env
LLM_PROVIDER=minimax
MINIMAX_API_KEY=你的_KEY
```

也可以不手写 `.env`，首次打开后在激活页配置：

```text
http://127.0.0.1:3721/activation
```

### 5.5 创建本地语音 Python 环境

```bash
cd "/你的路径/BaiLongma"
python3.11 -m venv .venv-whisper
source .venv-whisper/bin/activate
python -m pip install --upgrade pip setuptools wheel
```

安装 Whisper、SenseVoice 和声纹依赖：

```bash
pip install openai-whisper websockets numpy soundfile
pip install funasr modelscope huggingface_hub
pip install resemblyzer webrtcvad-wheels
```

如果 Apple Silicon 上安装 PyTorch 有问题，可单独安装：

```bash
pip install torch torchaudio
```

### 5.6 下载 SenseVoiceSmall 模型

推荐把模型放在项目目录：

```bash
mkdir -p models
```

方式一：使用 Hugging Face：

```bash
source .venv-whisper/bin/activate
huggingface-cli download FunAudioLLM/SenseVoiceSmall --local-dir models/SenseVoiceSmall
```

方式二：使用 ModelScope：

```bash
source .venv-whisper/bin/activate
modelscope download --model iic/SenseVoiceSmall --local_dir models/SenseVoiceSmall
```

下载完成后确认至少存在模型文件：

```bash
ls models/SenseVoiceSmall
```

也可以不预下载。首次启动 SenseVoice 时，程序会尝试从远端模型仓库加载，但预下载更稳定。

### 5.7 启动桌面版

```bash
npm start
```

启动后打开：

```text
http://127.0.0.1:3721/brain-ui
```

语音本地服务默认端口：

```text
ws://127.0.0.1:3723
```

### 5.8 推荐语音设置

在 Brain UI 设置页进入“语音识别”：

- 服务商：`本地`
- 本地模型：`SenseVoiceSmall`
- 识别语言：`中文`
- 启用唤醒词：开启
- 默认唤醒词：`小龙马，龙马，白龙马`
- 只响应我的声音：声纹录入成功后再开启
- 声纹严格度：先用 `0.55`
- 检测到人声时自动降低/暂停视频：开启
- 视频播放时启用空格按住说话：开启
- 启用系统回声消除 AEC：开启

声纹录入建议：

- 录 5 到 10 秒
- 离麦克风保持日常使用距离
- 用正常说话音量
- 说几句自然中文，不要只念一个词
- 先在安静环境录入
- 如果视频场景识别不稳定，后续可重新录入一版带轻微背景声的样本

## 7. 常见问题

### 7.1 语音服务启动失败

先确认 Python 环境存在：

```bash
ls .venv-whisper/bin/python
```

再手动启动 SenseVoice 服务看日志：

```bash
source .venv-whisper/bin/activate
python src/voice/sensevoice_server.py --model sensevoice-small --port 3723
```

### 7.2 SenseVoice 模型找不到

确认：

```bash
ls models/SenseVoiceSmall
```

如模型在别的位置，可设置环境变量：

```bash
export BAILONGMA_SENSEVOICE_MODEL_DIR="/你的模型路径/SenseVoiceSmall"
npm start
```

### 7.3 声纹录入后识别不到

先把声纹严格度降到 `0.55` 或 `0.50` 测试。
如果仍不稳定，删除旧声纹后重新录：

```bash
rm -f data/voiceprint.json
```

然后在设置页重新录入。

声纹不是唤醒词模型，它需要足够清晰的人声。视频声音太大时，建议同时开启视频降音、按住说话和 AEC。

### 7.4 播放视频时唤醒失败

优先检查：

- 视频音量是否过大
- Mac 麦克风输入音量是否过低
- 是否开启“检测到人声时自动降低/暂停视频”
- 是否开启“视频播放时启用空格按住说话”
- 是否开启 AEC

在特别吵的场景，推荐按住空格说话，这是最稳定的人机交互方式。

### 7.5 出现“我只想说了”等重复幻觉文本

当前版本已经在 Whisper 和 SenseVoice 服务里加入了过滤。若仍出现，优先：

- 提高触发阈值
- 开启唤醒词
- 开启声纹
- 降低视频音量
- 使用 SenseVoiceSmall 而不是 Whisper

## 8. 当前版本限制

- SenseVoiceSmall 是本地 ASR，不是本地大模型助手。
- 唤醒词目前仍是软件侧文本/音频链路判断，不等同于 ESP32 WakeNet 这类专用 KWS。
- 声纹目前使用 `resemblyzer`，适合个人桌面辅助，但还不是工业级 3D-Speaker/ECAPA 声纹系统。
- 视频抗干扰依赖浏览器/系统可用的 AEC、音量控制和前端检测逻辑，极端噪声下仍建议使用按住说话。
- macOS 打包配置还没有作为主路径完善；当前推荐源码运行 Electron。

## 9. 下一步建议

推荐下一版继续做：

- 增加 2 到 3 秒 pre-roll 环形音频缓存
