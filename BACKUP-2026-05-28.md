# BaiLongma v0.3.0 里程碑备份说明（2026-05-28）

## 版本定位

- 版本号：`v0.3.0`
- 维护仓库：`https://github.com/xiaoguiwucan/BaiLongma`
- 里程碑：微信群助手可用版。
- 核心结果：用户已经测试确认，微信群里 @ 扫码登录的微信账号后可以正常回复。

## 本版本包含的关键能力

1. 微信群助手独立设置菜单。
2. Wechaty 扫码登录/恢复登录。
3. 获取真实微信群列表并支持勾选多个群组。
4. 只有群里 @ 当前登录账号才调用大模型，普通群消息不主动打扰。
5. 回复时 @ 原提问人，并避免把内部联系人 ID 直接显示到群里。
6. Honcho 群知识库配置入口：每个微信群独立 session/peer。
7. 群指令安全守卫：拒绝危险电脑/账号操作。
8. Mac Electron 启动脚本与 Dock 显示修复。
9. 火山/豆包 ASR、TTS 自回声、整段播报等语音链路同步修复。

## 从 GitHub 恢复部署

```bash
git clone https://github.com/xiaoguiwucan/BaiLongma.git
cd BaiLongma
git checkout v0.3.0
npm install

# 推荐：Mac 桌面启动
./start-jarvis.command

# 或者普通 Electron 启动
npm start
```

启动后打开设置页：

1. 进入“微信群助手”。
2. 点击登录/恢复微信，按二维码扫码。
3. 等待显示已登录/在线状态。
4. 刷新或等待自动获取群组列表。
5. 勾选需要接入的群组，例如“值班群”“PT站看片狂魔小群”。
6. 点击“保存并生效”。
7. 到微信群里 @ 刚扫码登录的微信账号进行测试。

## Honcho 群知识库部署说明

本版本只上传 Honcho 接入代码和设置入口，不上传任何私有群聊数据。默认配置：

- Honcho Base URL：`http://127.0.0.1:8018`
- App / Knowledge Base：`bailongma-wechat-memory`

如果要启用本地 Honcho，请先按 Honcho 官方方式启动服务，再在设置页填写/确认地址。Honcho 未启动时：

- 微信群 @ 回复仍可正常工作；
- 群知识库状态会显示不可用；
- 按用户要求，不启用本地兜底记忆，不偷偷写入其他本地库。

## 验证清单

发布前应至少验证：

```bash
node --check src/social/wechaty-duty-group.js
node --check src/api.js
node --check src/config.js
node --check src/social/wechat-group-memory.js
node --check src/social/wechat-groups.js
node --check src/social/wechat-command-guard.js
node --check src/social/dispatch.js
node --check src/social/index.js
node --check src/ui/brain-ui/app.js
node --check src/ui/brain-ui/app-shell.js
node --check electron/main.cjs
```

人工验证：

1. 扫码登录后设置页显示真实登录状态。
2. 退出设置页再进入，状态和群组列表不应消失。
3. 保存群组选择后 Wechaty 不应掉线。
4. 在已勾选群里 @ 登录账号，应调用大模型回复。
5. 未勾选群或未 @ 的普通消息不应主动回复。
6. 危险指令应被安全守卫拒绝。

## 回滚方法

如需回滚到上一个语音状态机版本：

```bash
git checkout v0.2.0
npm install
npm start
```

如只是微信群助手异常，优先尝试：

1. 在设置页停止微信群助手。
2. 重新点击登录/恢复微信。
3. 重新扫码并等待群列表恢复。
4. 重新保存群组选择。
