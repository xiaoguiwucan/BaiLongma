# BaiLongma v0.4.7 备份说明（2026-05-28）

## 版本定位

- 版本号：`v0.4.7`
- 维护仓库：`https://github.com/xiaoguiwucan/BaiLongma`
- 里程碑：微信群助手可用版 + Honcho 群记忆管理 + 性格预设 + 安全黑名单 + 网络梗/称呼记忆 + 网络图片发送安全边界 + 全量群统计/排行榜/定时总结 + 手动选择统计群组修复 + 聊天记录库可视化/导入导出 + 昵称强制刷新修复 + 多群统计区分 + 多人 @ 排队 + 管理员模式 + LLM 多模型池/自动故障切换版，并修复微信群 @ 回复错人。
- 核心结果：用户已经测试确认，微信群里 @ 扫码登录的微信账号后可以正常回复；本版本在全量群消息统计基础上新增“微信群聊天记录库”，可以按群查看所有已入库聊天记录、完整时间、消息总数、成员昵称、类型/关键词/时间筛选，并支持 JSON/CSV 导出和 JSON 导入；新收到的图片/表情/音视频等媒体会尝试保存到本机数据目录并可在记录库预览/备份；本版本进一步修复微信群昵称仍显示“未知成员”的问题，支持手动刷新昵称并按群名合并重新登录后的新旧 room_id；v0.4.5 在此基础上新增多人 @ 排队回复、多群统计总览、排行榜来源群标注、精确 sender_id 管理员模式和榜单自动刷新；v0.4.6 新增 LLM 多模型池与自动故障切换，在当前模型没额度/限流/不可用时自动切到备用模型；v0.4.7 修复微信群 @ 回复错人，按真实提问人精确 @ 回去。

## 本版本包含的关键能力

1. 微信群助手独立设置菜单。
2. Wechaty 扫码登录/恢复登录。
3. 获取真实微信群列表并支持勾选多个群组。
4. 只有群里 @ 当前登录账号才调用大模型，普通群消息不主动打扰。
5. 回复时 @ 原提问人，并避免把内部联系人 ID 直接显示到群里。
6. Honcho 群知识库配置入口：每个微信群独立 session/peer。
7. Honcho 群记忆管理 UI：按群查看原始消息、自动摘要、长期结论；支持手动添加长期记忆、删除结论、清空本群 session。
8. 微信群助手性格设定：支持手动编辑提示词，并内置 3 个一键预设。
9. 群指令安全守卫：拒绝查看/列出本机文件、读取凭证、执行命令、控制电脑、账号资金等高危请求，设置页展示 19 类黑名单规则。
10. ClawBot 群聊路径也接入安全守卫，避免旁路绕过。
11. 网络梗理解：v我50 / vw50 / 疯狂星期四等会按中文群聊梗处理。
12. 群媒体边界：允许公开网络图片/表情包链接，禁止本机文件、桌面文件、截图、相册外发。
13. 显式称呼/身份记忆：群成员说“以后叫我大哥 / 我是你大哥 / 我叫xxx”会即时写入 Honcho 本群长期记忆。
14. Honcho 记忆详情分开展示群组长期记忆和成员长期记忆，成员记忆仅在当前群内可用并与群组记忆共同匹配。
15. 微信群助手性格设置具备独立保存按钮和明显生效状态。
16. Mac Electron 启动脚本与 Dock 显示修复。
17. 火山/豆包 ASR、TTS 自回声、整段播报等语音链路同步修复。
18. 修复微信群偶发回复英文内部协议文本的问题，内部工具执行状态不会再发到群里。
19. 微信群全量活动统计：记录所有已接入群的文字、图片、表情、链接、小程序等消息。
20. 群排行榜：发言榜、发图榜、表情榜、链接榜、装逼榜（启发式娱乐统计）。
21. 定时总结：支持阶段总结和每日 00:00 群聊日报，自动去重避免重复发送。
22. 设置页新增“群统计与定时总结”面板，可以手动保存配置、刷新统计、立即发送本群总结.
23. 群统计排行榜优先显示微信群昵称/备注/微信昵称，不再把 `@a5a383...` 这类 WeChaty 内部 ID 直接显示出来。
24. Wechaty 接入群或收到消息后会后台刷新群成员昵称，并自动回填旧统计表里的内部 ID。
25. 微信群聊天记录库：按当前群查看本机 SQLite 已入库消息总数、完整时间、成员昵称、内容、图片/表情/链接/装逼统计。
26. 聊天记录筛选：支持开始时间、结束时间、消息类型、关键词检索和分页加载更多。
27. 聊天记录导出：支持 JSON 完整备份和 CSV 表格导出。
28. 聊天记录导入：支持 JSON 恢复，重复记录会跳过；JSON 中包含的媒体文件会恢复到本机数据目录。
29. 媒体留存：新收到的图片、表情、音频、视频、附件会尝试保存到 `data/wechat-media/`，设置页可以预览图片/播放音视频。
30. 媒体安全：记录库媒体预览只允许读取已入库相对路径，不能借此读取任意本机文件。
31. 昵称强制刷新：直接调用 wechat4u `batchGetContact` 拉取群成员昵称，解决 Wechaty 默认 API 只能拿到“未知成员”的问题。
32. room_id 变化合并：重新扫码后微信群 room_id 变化时，统计和聊天记录按群名合并，不再拆成两个群。
33. 聊天记录库 UI 优化：主查询按钮、今天快捷按钮、刷新昵称按钮、清晰筛选输入框和用途说明。
34. 多人 @ 排队：微信群 @ 消息不再按同群覆盖旧消息，也不会打断正在回复的用户消息，按到达顺序逐条处理。
35. 多群统计总览：统计页可切换“当前群 / 已选统计群总览”，多群排行榜每行显示来源群。
36. 管理员模式：只按 Wechaty sender_id 精确识别管理员，可从成员 ID 列表点选；管理员 @ 可跳过微信群黑名单，普通成员仍严格拦截。
37. 榜单自动刷新：设置页停留在微信群助手时每 12 秒刷新统计榜单，保存统计群后立即刷新。
38. LLM 多模型池：设置页可保存多个模型 profile，支持名称、Provider、模型、API Key/自定义端点、启停、排序、编辑、删除和设为当前。
39. LLM 自动故障切换：额度不足、余额不足、限流、认证失败、模型不可用、服务端 5xx 或网络超时时，尚未输出内容前自动切换备用模型，并记录失败冷却。
40. LLM 安全展示：设置页和 `/settings` 只显示 Key 是否已配置与尾号，不返回明文 API Key。
41. 微信群 @ 修复：每条群消息生成独立回复目标，按当前提问人 sender_id / sender_name 精确 @，不再误 @ 管理员或上一位成员。


## 从 GitHub 恢复部署

```bash
git clone https://github.com/xiaoguiwucan/BaiLongma.git
cd BaiLongma
git checkout v0.4.6
npm install

# 推荐：Mac 桌面启动
./start-jarvis.command

# 或者普通 Electron 启动
npm start
```

启动后打开设置页：

1. 进入“微信群助手”。
2. 点击登录/恢复微信，按二维码扫码。
3. 等待显示已登录/在线状态。
4. 刷新或等待自动获取群组列表。
5. 勾选需要接入的群组，例如“值班群”“PT站看片狂魔小群”。
6. 在「微信群助手性格设定」选择预设：主人数字分身、技术值班助手、幽默社交助手；也可以手动微调文本。
7. 点击“保存并生效”。
8. 到微信群里 @ 刚扫码登录的微信账号进行测试。



## v0.4.6 本次更新内容

- 新增 LLM 模型池：`设置 -> LLM 模型` 可添加多个模型配置，不再只能保存一个 Provider/模型。
- 每个模型配置支持自定义名称、Provider、模型、API Key、自定义 Base URL、启用/关闭、编辑、删除、上移/下移优先级、设为当前。
- 自动故障切换默认开启：当前模型没额度、余额不足、限流、认证失败、模型不可用、5xx 或网络超时时，会按优先级切换下一个启用模型。
- 只在回答尚未输出内容时切换，避免回复重复、语音播报断裂或内容拼接错乱。
- 失败模型会写入 `lastError/lastFailedAt/cooldownUntil`，冷却期间自动跳过，避免每轮都打到没额度的模型。
- 旧版单模型配置会自动迁移为模型池第一项，原 `/activate` 和 `/settings/model` 仍兼容。
- 新增 API：`POST /settings/llm-profile`、`POST /settings/llm-profile/select`、`POST /settings/llm-profile/delete`、`POST /settings/llm-failover`。
- `/settings` 返回 `llm.profiles`、`llm.activeProfileId`、`llm.failover`，但不会返回明文 API Key。
- Brain UI 会在模型切换时显示“当前模型不可用，正在无缝切换到备用模型”的状态提示。

使用方式：更新重启后进入 `设置 -> LLM 模型`，至少添加两个启用模型。把常用模型放在最上方，把备用模型排在后面；保持“额度不足/限流时自动切换备用模型”开启。

## v0.4.5 本次更新内容

- 新增“按群拆分”概览，多个群的统计不会混在一起看不懂。
- 新增管理员模式设置区：启用开关、管理员 sender_id 输入框、最近成员 ID 列表、一键加入管理员。
- 管理员识别只使用 Wechaty `sender_id` 精确匹配，不使用昵称、备注或群里自称。

- 新增成员 ID API：`GET /social/wechat-groups/members`。
- 榜单在设置页打开并停留在“微信群助手”时每 12 秒自动刷新；保存统计群组后立即刷新。

使用方式：更新重启后进入 `设置 -> 微信群助手`。需要管理员时先点击“刷新成员 ID”，从成员列表点选自己的微信 sender_id，再启用“管理员模式”并保存。多群统计在“群统计与定时总结”里先勾选群并保存，再把查看范围切到“已选统计群总览”。

## v0.3.1 补丁说明：微信群 @ 不再绑定昵称

### 问题

用户在微信群里 @ 助手后，程序能收到 Wechaty 的 `mention=true`，也能正确 @ 原提问人回复，但大模型会根据文本里的展示昵称再次判断“是不是叫我”。当群昵称/备注/微信名显示为“小风”“风”等不同名称时，模型误判并回复“没叫我，跳过”。

### 修复

- 触发入口只信任 Wechaty `message.mentionSelf()` 元数据。
- 不再绑定“前夜 / 小白龙 / 贾维斯 / 小风”等任何固定文本。
- 群提示词明确告诉模型：Wechaty 已确认 @ 当前登录账号，禁止再判断昵称。
- `send_message` 工具和 fallback 都会拦截“没叫我/跳过/无需回应”这类错误回复。
- 只有 `send_message` 真正投递成功才算本轮已回复。

### 恢复/部署

```bash
git checkout v0.3.1
npm install
./start-jarvis.command
```

本补丁不需要重新配置 Honcho，不上传任何群聊数据。


## v0.3.2 补丁说明：Wechaty 登录态持久化

### 结论

正常情况下白龙马不应该每次重启都要求重新扫码。重启后会优先复用 `~/Library/Application Support/Bailongma/wechaty-duty-group.memory-card.json` 中的 Wechaty 登录态。

### 仍需重新扫码的情况

- 首次使用或登录态文件还没有生成。
- 微信服务端判定 Web 登录态过期。
- 手机微信里手动退出/踢掉网页微信登录。
- 更换微信号、清理 userData、删除 `wechaty-duty-group.memory-card.json`。
- Wechaty/wechat4u 上游登录接口被微信限制。

### 本补丁修复

- root `MemoryCard` 显式传给 `WechatyBuilder`。
- 登录态固定写入 userData，不再依赖进程 cwd。
- 正常 stop/restart 不删除 `PUPPET-WECHAT4U`。
- `.gitignore` 忽略 `*.memory-card.json`，避免登录态上传。


## v0.3.3 补丁说明：关闭窗口即彻底退出

### 问题

用户点击 macOS 窗口关闭按钮后，Bailongma 主窗口消失，但菜单栏/托盘图标仍存在，后台服务也继续运行。原因是 Electron 主进程原本把 `close` 事件拦截后执行 `mainWindow.hide()`，设计成“关闭窗口=隐藏到菜单栏”。

### 修复

- 点击主窗口关闭按钮时设置 `app.isQuiting = true` 并调用 `app.quit()`。
- 新增 `before-quit` 统一维护退出标记。
- `window-all-closed` 改为调用 `app.quit()`，最后窗口关闭后不再后台常驻。
- 菜单栏“退出”仍保留，作为程序运行时的显式退出入口。

### 恢复/部署

```bash
git checkout v0.3.3
npm install
./start-jarvis.command
```

### 注意

如果当前旧版本已经隐藏到菜单栏，需要先从菜单栏 Bailongma 图标选择“退出”，或者用活动监视器结束旧进程，再启动 v0.3.3。


## v0.3.4 补丁说明：微信群助手真实状态与强制重新扫码

### 问题

设置页显示“已登录/群列表已刷新”，但实际群里 @ 登录账号没有任何回复。深排发现后端同时存在 `logged_in/online:true` 和 `400 != 400`、群列表无法真实刷新等异常，UI 展示的是历史群列表快照，不代表当前 Wechaty 真的能收消息。

### 修复

- `online` 改为真实在线判定：必须当前进程 connected、解析到目标群，并且最近真实刷新群列表或收到群消息。
- 旧群列表标记为 `rooms_stale`，UI 显示“缓存”，不再叫“真实已选”。
- `/rooms` 没拿到真实群列表时返回失败和提示，不再误报刷新成功。
- 设置页新增“强制重新扫码”，会清空 Wechaty 登录态并生成新二维码。
- 修复 Wechaty MemoryCard：改用 `WechatyBuilder.build({ name: WECHATY_MEMORY_NAME, puppet })`，确保二维码链路可用。

### 恢复/部署

```bash

./start-jarvis.command
```

1. 如果显示未确认在线或缓存群列表，点击“强制重新扫码”。
2. 用要接入群聊的微信扫描二维码。
3. 扫码成功后刷新真实群列表，勾选需要接入的群。
4. 点击“保存并生效”。
5. 在群里 @ 该微信账号测试。
